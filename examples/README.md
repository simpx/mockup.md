# Mockup Examples

This directory contains example Mockup (`.mu`) presentations demonstrating various use cases.

## Directory Structure

Each example is in its own directory with:
- Source `.mu` file
- `README.md` describing the example
- Subdirectories for AI-generated images (to be added)

## Examples

### [intro](./intro/)
**Product Introduction Presentation**
- 8 slides introducing Mockup itself
- Demonstrates: cover slides, problem/solution, workflow, split layouts, grid layouts
- Best for: Learning the basics of Mockup syntax

### [tech-rfc](./tech-rfc/)
**Technical RFC Presentation**
- 6 slides proposing a database infrastructure upgrade
- Demonstrates: architecture diagrams, line graphs, timeline visualization, emoji annotations
- Best for: Engineering proposals and technical documentation

### [thesis-defense](./thesis-defense/)
**Academic Thesis Defense**
- 7 slides for a machine learning paper defense
- Demonstrates: academic formatting, model architecture diagrams, quantitative results, comparison tables
- Best for: Academic presentations and research talks

### [mapreduce](./mapreduce/)
**Classic Paper Presentation**
- 15 slides presenting the MapReduce paper (Google, 2004)
- Demonstrates: teaching content, sequence diagrams, data flow visualization, performance charts
- Best for: Paper presentations, educational talks, comprehensive technical content

## Generated Images

Each example directory will contain subdirectories for images generated by different AI models:

```
examples/
├── intro/
│   ├── intro.mu
│   ├── README.md
│   ├── claude-sonnet-4/
│   │   ├── slide-01.png
│   │   ├── slide-02.png
│   │   └── ...
│   ├── gpt-4/
│   │   └── ...
│   └── other-models/
│       └── ...
```

This structure allows for:
- Comparing output quality across different AI models
- Tracking improvements as models evolve
- Providing reference images for users

## Using These Examples

1. **Learn the syntax**: Read the `.mu` files to understand Mockup syntax
2. **Generate your own**: Use these as templates for your presentations
3. **Compare outputs**: Check the generated images to see AI interpretation quality
4. **Contribute**: Generate images with your preferred AI model and contribute them back

## Contributing Generated Images

If you generate presentation images from these examples:

1. Create a subdirectory named after the AI model (e.g., `claude-opus-4/`, `gemini-pro/`)
2. Include:
   - Individual slide images (PNG format, 1920x1080 recommended)
   - Optional: Full presentation PDF
   - `metadata.json` with generation details:
     ```json
     {
       "model": "claude-sonnet-4.5",
       "date": "2026-02-01",
       "prompt": "Generate presentation images from this .mu file",
       "notes": "Any special observations or settings used"
     }
     ```
3. Submit a pull request

This helps the community understand how different AI models interpret and render Mockup layouts.
